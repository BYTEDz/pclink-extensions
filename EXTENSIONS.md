### Technical Specifications

- **Framework**: `PCLink Extension Core v2.0`
- **Frontend**: Utilizes the **PCLink Theme SDK** for user interface consistency.
- **Backend**: Asynchronous Python runtime with isolated dependency execution.

### Virtual Environments (v2.0)

Extensions can specify unique dependencies using a standard `requirements.txt` file, which the loader installs into an isolated virtual environment.

- Prevents dependency conflicts between individual extensions and the core server.
- Allows extensions to target specific package versions independently.
- Core server libraries are isolated from the extension runtime environments.

To enable virtual environment support, add the configuration parameter to `extension.yaml`:

```yaml
requirements_file: requirements.txt  # Path relative to extension root
```

Add the target packages to a `requirements.txt` file in your extension folder:

```text
numpy>=1.24.0
requests>=2.28.0
```

The system initializes the virtual environment during the initial extension load process.

### Asynchronous Execution (v2.0)

Extensions can implement asynchronous `initialize()` and `cleanup()` routines:

```python
from pclink.core.extension_base import ExtensionBase

class Extension(ExtensionBase):
    async def initialize(self) -> bool:
        # Asynchronous initialization routine
        await self._setup_websocket()
        return True

    async def cleanup(self):
        # Asynchronous cleanup routine
        await self._close_connections()
```

The extension manager detects and executes both synchronous and asynchronous implementations, ensuring backward compatibility with legacy extensions.

### Platform Requirements

- **Windows**: Integrates via WinAPI and COM interfaces.
- **Linux**: Supported on distributions including Ubuntu, Fedora, and Arch.
- **macOS**: Supported on macOS environments.

> [!IMPORTANT]  
> All listed extensions adhere to the PCLink Security Model. Hardware permission requests are declared within each extension manifest.
